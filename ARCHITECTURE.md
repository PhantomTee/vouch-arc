# Vouch Architecture

A2A work marketplace with on-chain USDC escrow, reputation, and x402 payment settlement on Arc (Circle's USDC-native L1).

## Components

### `src/market-server.js` — Marketplace HTTP server

The main entry point (`npm run market`). Serves both the browser UI and a JSON API.

Responsibilities:
- Host UI pages generated by `src/pages.js`
- Run jobs via `ClientAgent` (async, returns a polling token)
- Expose discovery API (`/api/agents/discover`, `/.well-known/agent-card.json`)
- Serve Prometheus metrics at `/GET /metrics`
- Persist job outcomes to `jobs.json` and archive to `data/jobs.jsonl`
- Log structured JSON to stderr via `src/logger.js`

### `src/agents.js` — Agent logic

**`WorkerAgent`**: wraps a solver function (built-in or remote), builds an A2A AgentCard, and delivers job artifacts.

**`ClientAgent`**: discovers workers via `AgentRegistry`, opens on-chain escrow, receives a delivery, verifies it with `src/verification.js`, then calls `completeEscrow` (paid) or `raiseDispute`.

**`solvers`**: built-in implementations — `honestCoder` (correct addition), `buggyCoder` (wrong answer), `honestInference` (challenge-aware stub).

### `src/chain.js` — On-chain escrow

`Market` class wraps the `WorkEscrow` Solidity contract via ethers.js v6:

| Method | Description |
|---|---|
| `createEscrow(provider, amountUsdc, deadline)` | Lock USDC for a job |
| `completeEscrow(id)` | Release to worker on success |
| `raiseDispute(id)` | Open a dispute |
| `resolveDispute(id, providerWins)` | Settle a dispute (arbiter only) |
| `reputationScore(address)` | Read on-chain reputation integer |
| `scanSettlements()` | Scan event logs for paid/disputed history |

### `src/verification.js` — Delivery verification

Two strategies, selected by `job.kind`:

**`verifyCode(artifact, spec)`**
1. Security scan — rejects `eval`, `new Function`, `require(fs)`, `process.exit`, etc.
2. Sandbox execution via `node:vm` with console capture
3. Test-case comparison with type checking
4. Performance scoring (0–100)

**`verifyInference(delivery, spec)`**
1. Freshness check — attestation `timestamp` must be < 60 seconds old
2. Integrity check — SHA-256 digest + secret-keyed signature must match
3. Known-answer challenge — provider must answer a question the verifier already knows

### `src/arbitration.js` — Dispute resolution

`disputeResolution(verifyResult)` returns one of two outcomes:
- `auto_worker_loses` — verification failed; worker clearly delivered bad work
- `human_review` — verification passed but client disputes; needs operator attention

All dispute events and completed jobs are persisted as append-only JSONL:
- `data/disputes.jsonl`
- `data/jobs.jsonl`

### `src/a2a.js` — A2A AgentCard v0.2

Builds fully-compliant agent cards with:
`id`, `type`, `capabilities.artifacts`, `reputation`, `pricingTerms`, `availabilityStatus`, `discoveryEndpoint`, per-skill `proficiency`, and x402 `PaymentRequirements` on Arc.

### `src/registry.js` — Agent discovery

`AgentRegistry` holds registered workers and provides `discover(skill, opts)` — filters by skill, maximum price, and x402 support. Used by `ClientAgent` to pick the best worker.

### `src/roster.js` — Worker roster

Combines two sources:
- `src/agents.json` — built-in workers (committed to repo)
- `registry.json` — operator-registered workers (written by UI or `npm run worker`)

### `src/coordinator.js` — Live worker directory

Lightweight HTTP server (`npm run coordinator`) that tracks worker nodes announcing themselves. Workers `POST /register` on startup and `POST /heartbeat` every 10 seconds. Registrations expire after 40 seconds of silence.

### `src/worker-node.js` — Standalone worker

A separate process (`npm run worker:serve`) that serves `POST /deliver`, registers with the coordinator on startup, and sends heartbeats. Uses a solver from `src/agents.js`.

### `src/autonomousDriver.js` — Continuous loop

`runAutonomousLoop({ client, jobQueue, pollIntervalMs })` polls for jobs every N ms and runs the full `client.runJob()` cycle. Handles SIGTERM gracefully — finishes the current job before exiting.

### `src/loop.js` — Daemon entry point (`npm start`)

Wires up `Market`, `ClientAgent`, a round-robin demo job queue, and `runAutonomousLoop`. Emits Prometheus metrics and archives settled jobs on each cycle.

### `src/logger.js` — Structured logging

Pino-compatible JSON logger writing to stderr. Set `LOG_LEVEL=debug|info|warn|error` (default: `info`). Each line: `{ level, time, pid, hostname, msg, ...fields }`.

### `src/metrics.js` — Prometheus metrics

In-process counters and histograms in Prometheus text format. No external dependencies. Served at `GET /metrics`.

## Smart contract — `contracts/WorkEscrow.sol`

Deployed on Arc Testnet. Hardhat 2 + OpenZeppelin.

```
WorkEscrow
├── createEscrow(provider, amount, deadline) → escrowId
├── completeEscrow(id)           — releases funds to provider; reputation++
├── raiseDispute(id)             — client flags bad delivery
├── resolveDispute(id, bool)     — arbiter settles; reputation-- on loss
├── reputationScore(address)     — public int64 score
└── getEscrow(id)                → { client, provider, amount, status, … }

Status enum: Open → Released | Disputed → Settled
```

ReentrancyGuard + SafeERC20 for USDC transfers. USDC address on Arc: `0x3600000000000000000000000000000000000000`.

## Data flow — single job

```
Client POST /api/post-job
  │
  ├─ ClientAgent discovers worker via AgentRegistry
  │
  ├─ market.ensureApproval(budget)           ← USDC allowance on Arc
  │
  ├─ market.createEscrow(worker, amount, 1h) ← on-chain escrow created
  │    inc("vouch_escrows_created_total")
  │
  ├─ worker.deliver(job)                     ← WorkerAgent runs solver
  │
  ├─ verification.verify(job, delivery)      ← securityScan + vm sandbox
  │                                            or attestation + challenge
  │
  ├─ [passed] market.completeEscrow(id)      ← USDC released to worker
  │    inc("vouch_escrows_settled_total")    ← reputation++ on-chain
  │    archiveJob(...)                       ← data/jobs.jsonl
  │
  └─ [failed] market.raiseDispute(id)        ← escrow locked pending review
       inc("vouch_disputes_raised_total")
       disputeResolution(result)             ← auto_worker_loses | human_review
       logDispute(...)                       ← data/disputes.jsonl
```
